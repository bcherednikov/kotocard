## Future Feature List (Product Growth)

> Черновик идей для развития продукта.  
> Не является обязательным бэклогом — это «куда можем расти дальше».

---

### 1. Универсальная модель пользователей и групп

- **Отказ от жёсткой схемы «родитель–ребёнок»**  
  Любой пользователь может создавать свои карточки и учить их сам.  
  Роль «родителя» превращается в просто одного из пользователей, который может назначать наборы другим (детям, ученикам и т.д.).

- **Группы пользователей**  
  - Типы групп: класс, семья, друзья, учебная группа и т.п.  
  - Пользователь может состоять в нескольких группах.  
  - Внутри группы могут быть общие наборы карточек, доступные всем участникам.

> **NB:** Эту модель нужно дополнительно «прожарить» архитектурно: роли, права, владение контентом, доступ к статистике и т.п.

---

### 2. Шаринг и переиспользование наборов карточек

- **Общий каталог наборов**  
  - Любой пользователь видит все публичные наборы карточек.  
  - Возможность «подписаться» на набор и добавить его себе.

- **Назначение наборов другим пользователям**  
  - Из общего каталога или из своих наборов можно «закинуть» набор ребёнку / ученику / другу.  
  - Поддержка назначения набора сразу нескольким детям / участникам группы.

---

### 3. Персональное назначение наборов детям / ученикам

- **Учебный план на пользователя**  
  - Для каждого ребёнка выбираются конкретные наборы, которые он сейчас изучает.  
  - Родитель и сам ребёнок (если разрешено) могут добавлять/убирать наборы из его плана.  
  - Отдельный экран «что учит этот ребёнок сейчас».

---

### 4. Улучшенный режим повторения

- **Режимы практики**
  - **Flashcards**: классическое переворачивание карточки (слово → перевод).  
  - **Listening**: проигрываем только аудио — пользователь вспоминает слово/перевод.  
  - **Writing**: пользователь печатает перевод/слово.  
  - **Speaking**: пользователь произносит слово вслух (в перспективе — с распознаванием речи).

- **Фильтры для сессий повторения**
  - Только новые карточки.  
  - Только сложные карточки (с высоким количеством ошибок).  
  - Все карточки выбранного набора / пользователя.  
  - Карточки, не повторявшиеся X дней.

- **График повторений**
  - Отображение, когда и какие карточки были повторены.  
  - Визуализация прогресса по дням/неделям.

---

### 5. Progress Tracking и награды

- **Дневная статистика**
  - Сколько карточек пользователь/ребёнок повторил за день.  
  - Серии дней (streak), визуализация по календарю.

- **Недельные цели и награды**
  - Цель: заполнить неделю по X карточек в день.  
  - При выполнении цели — награда (бейджи, ачивки, внутренняя валюта, стикеры и т.п.).

---

### 6. Групповая динамика и соревнование

- **Лидерборды по группе**
  - Кто сколько карточек выучил.  
  - Сколько дней подряд занимается каждый участник.  
  - Сравнение прогресса внутри класса, семьи, дружественной группы.

- **Мотивационные механики**
  - Совместные челленджи (например: «вся группа учит по 20 карточек в день неделю подряд»).  
  - Публичные ачивки внутри группы.

---

### 7. Интервальное повторение (Spaced Repetition)

- **Базовый алгоритм**
  - День 1: учу слово.  
  - День 2: повтор; если ответ правильный → следующая проверка через 4 дня.  
  - День 6: повтор; если правильный → через 7 дней.  
  - День 13: повтор; если правильный → через 14 дней.  
  - Далее — увеличение интервалов.

- **Упрощённая версия для MVP 2.0**
  - Фиксированные интервалы: 1 день, 3 дня, 7 дней, 14 дней, 30 дней.  
  - При ошибке интервал сбрасывается на начальный.

---

### 8. Улучшенный AUDIO (TTS)

- **Более «человечный» TTS для RU/EN**
  - Поиск и интеграция бесплатной или условно-бесплатной модели TTS с естественным голосом (русский и английский).  
  - Поддержка озвучивания отдельных слов и примеров использования.  
  - В перспективе — выбор голоса/акцента.

---

### 9. Генерация короткого текста из выученных слов

- **Идея**  
  Сервис генерирует короткий связный текст (2–3 предложения или короткий рассказ), используя слова из карточек, которые пользователь выучил, или из всего доступного словаря.

- **Методика по объёму**
  - Мало слов → 2–3 предложения.  
  - Больше слов → короткий рассказ.  
  - Неизвестные слова добавляются по минимуму — только связки для смысла (артикли, предлоги, союзы, базовые глаголы и т.п.).

- **Уровень сложности (настраивается)**
  - Влияет на количество «связующих» неизвестных слов и на грамматику.  
  - **Базовый уровень:** настоящее время (Present Simple и т.п.), минимум связок, короткие предложения.  
  - Более высокие уровни: прошлое/будущее время, более сложные конструкции, больше связок.

- **Источник слов**
  - Только выученные карточки (по прогрессу пользователя).  
  - Или весь словарь набора / нескольких наборов (опция «из всего словаря»).

---

### 10. Статус изучения набора

- **По каждому набору показывать:** сколько карточек выучил, сколько ещё не выучено (новые / в процессе).
- Отдельная сводка по набору: выучено / всего, процент, при желании — разбивка по направлению (RU→EN, EN→RU).
- Чтобы пользователь и родитель видели прогресс по набору «сразу в цифрах».
